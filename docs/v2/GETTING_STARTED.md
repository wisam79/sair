# Sair v2 - Getting Started Guide 🚀

This guide provides instructions for setting up the Sair v2 development environment on your local machine.

---

## Prerequisites

Before beginning, ensure you have the following installed:

1. **Node.js**: Version `22.0.0` or higher (verified with `node -v`).
2. **PNPM**: Package manager, version `10.0.0` or higher (`npm install -g pnpm`).
3. **Supabase CLI**: Latest version for local database and edge function development.
4. **Docker Desktop**: Required by the Supabase CLI to spin up local database and auth containers.
5. **Git**: For version control.

---

## 1. Project Installation

Clone the repository and install all workspace dependencies:

```bash
git clone <your-repo-url> sair
cd sair
pnpm install
```

---

## 2. Environment Configurations

Sair v2 manages configurations using localized `.env` files.

Copy the example environment file to generate your configuration:

```bash
cp .env.example .env
```

Ensure your root `.env` (or local application environment files) contain the following variables:

### Database & Client Connections

```env
DATABASE_URL="postgres://postgres:postgres@127.0.0.1:54322/postgres"
NEXT_PUBLIC_SUPABASE_URL="http://127.0.0.1:54321"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your-local-anon-key"
```

### Mobile App (`apps/mobile/.env`)

Create an `.env` file in the mobile directory:

```env
EXPO_PUBLIC_SUPABASE_URL="http://127.0.0.1:54321"
EXPO_PUBLIC_SUPABASE_ANON_KEY="your-local-anon-key"
```

### Admin App (`apps/admin/.env.local`)

Create an `.env.local` file in the admin directory:

```env
NEXT_PUBLIC_SUPABASE_URL="http://127.0.0.1:54321"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your-local-anon-key"
DATABASE_URL="postgres://postgres:postgres@127.0.0.1:54322/postgres"
```

---

## 3. Database Bootstrapping & Seeding

1. Make sure Docker is running on your machine.
2. Initialize and start the local Supabase container ecosystem:
   ```bash
   supabase start
   ```
3. Reset the database to run all migrations in `supabase/migrations/` and apply the test seed data (applied from `supabase/seed.sql`):
   ```bash
   supabase db reset
   ```

---

## 4. Running the Applications

Start all monorepo applications in development mode simultaneously from the root directory:

```bash
pnpm dev
```

Alternatively, you can run individual projects using pnpm filters:

### Running the Admin Dashboard (Refine + Next.js)

```bash
pnpm --filter admin-dashboard dev
```

The dashboard will be accessible at `http://localhost:3000`.

### Running the Mobile Application (Expo + React Native)

```bash
pnpm --filter mobile-app start
```

Press `a` to run the app on an Android Emulator, `i` for iOS Simulator, or scan the QR code using the Expo Go application on your physical device.

---

## 5. Serving and Testing Edge Functions Locally

To run and debug Edge Functions (like `trip-engine` or `send-notification`) locally:

1. Serve the edge functions:
   ```bash
   supabase functions serve
   ```
2. Edge functions are served at `http://127.0.0.1:54321/functions/v1/`.

To test functions, you can invoke them via `curl` or Postman by passing the `Authorization` header containing a valid user JWT token generated by your local Supabase instance.
